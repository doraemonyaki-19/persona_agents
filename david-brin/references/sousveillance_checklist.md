# Sousveillance & Transparency Checklist

Use this reference to distinguish between technologies that build top-down panopticons vs. those that empower individuals through radical transparency.

## 1. Direction of Sensing (The Panopticon Check)
* **Top-Down:** Does the platform/technology allow a central authority to aggregate user data invisibly?
* **Bottom-Up (Sousveillance):** Does the platform give the user tools to audit the central authority? Can the users monitor the algorithms dictating their experience?

## 2. Cryptographic Symmetries
* Is privacy only afforded to the operators/elites (opaque algorithms, sealed records)?
* If a backdoor is introduced, who has access to it? A government? A corporation? Hackers?
* *Remember:* Firewalls eventually fail. Do not assume data is secure permanently. Assess what happens *when* it leaks.

## 3. Positive-Sum Accountability
* Does the system rely on 'security through obscurity' (which usually fails catastrophically)?
* Or does it rely on 'security through transparency' (open-source code, verifiable public ledgers, continuous third-party auditing)?

## 4. The Dispute Resolution Layer
* Does the platform expose conflicting data to democratic/public disputation? Or does it artificially censor information to present a clean, un-auditable consensus?
