# Exponential Modeling & S-Curve Diagnostic

Use this reference when evaluating a technology's trajectory to avoid linear forecasting errors.

## 1. Information Horizon Check
Does this technology primarily deal with information? (e.g., compute, genomics, bandwidth, neural network parameters).
* If yes: It is highly likely adhering to an exponential doubling curve.
* If no (e.g., physical transportation, raw material extraction): Is there a sub-component that *is* an information technology that will disrupt the whole?

## 2. Doubling Rate Extraction
1. Fetch historical milestones for the technology over the last 10-20 years.
2. Determine the metric for 'Price-Performance' (e.g., calculations per second per dollar, cost per sequenced genome).
3. Calculate the doubling rate (e.g., doublings every 18 months, or 10 months).

## 3. S-Curve Saturation Evaluation
Every paradigm eventually hits a physical limit (forming an S-Curve).
* Has the current paradigm reached its physical limit? (e.g., thermal limits on silicon).
* If yes, what is the emerging parallel paradigm (e.g., optical computing, 3D stacking, quantum) that will act as the next 'S-Curve' and continue the overarching exponential trend?

## 4. The Singularity / Convergence Factor
* How will this technology accelerate when fed into advanced AI systems? 
* If progress is currently driven by human researchers, what happens to the timeline when the research itself is automated?
